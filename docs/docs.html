Quickstart

Install from source, initialize local storage, start a session, record commands, add notes, and generate documentation.

cargo install --path .
runbookai init
runbookai start "Fix payment 500"
runbookai exec "npm test"
runbookai note --type root-cause "JWT secret missing in test env"
runbookai exec "npm run build"
runbookai stop
runbookai generate all

Generated Markdown files are written to docs/runbooks/. Session data is stored locally in .runbookai/.

Recommended workflow

1. Start before debuggingCreate a named session before asking an AI agent to investigate or modify code.
2. Record commandsUse runbookai exec for full output capture, or shell hooks for automatic command logging.
3. Add human notesRecord findings, decisions, risks, root causes, and failed attempts while the context is still fresh.
4. Generate artifactsCreate runbooks, changelogs, postmortems, PR notes, and next-agent briefs from the same session.

Command reference

Session lifecycle

runbookai init
runbookai start "Fix login 401 error"
runbookai status
runbookai stop

Capture commands and notes

runbookai exec "npm test"
runbookai exec "npm run build"
runbookai note --type finding "Login test fails with 401."
runbookai note --type root-cause "JWT secret is missing."
runbookai note --type decision "Validate env during bootstrap."
runbookai note --type risk "Production env names must not change."

Generate documentation

runbookai generate runbook
runbookai generate changelog
runbookai generate postmortem
runbookai generate pr
runbookai generate all

Export and search

runbookai export --format json
runbookai export --format markdown
runbookai search "JWT secret"

Shell integration

If you do not want to type runbookai exec for every command, enable shell hooks. Commands will be recorded automatically whenever a session is active.

# Zsh
source <(runbookai shell-hook zsh)

# Bash
source <(runbookai shell-hook bash)
# PowerShell
runbookai shell-hook powershell | Invoke-Expression

For full command output and error capture, runbookai exec is still recommended.

AI-assisted summaries

RunbookAI can generate AI-assisted summaries with OpenAI, Gemini, or local Ollama.

runbookai generate runbook --ai
runbookai generate pr --ai
runbookai generate all --ai
  • OPENAI_API_KEY → OpenAI, default model gpt-4o
  • GEMINI_API_KEY → Gemini, default model gemini-1.5-pro
  • No API key → local Ollama, default model llama3
RUNBOOKAI_MODEL=llama3.1 runbookai generate runbook --ai

MCP server

RunbookAI supports the Model Context Protocol so AI agents can query previous sessions directly.

runbookai mcp serve

Claude Desktop example:

{
  "mcpServers": {
    "runbookai": {
      "command": "runbookai",
      "args": ["mcp", "serve"]
    }
  }
}
Toolsearch_sessions searches through old sessions.
Resourcerunbook://sessions/<id> reads full session data as JSON.

Custom templates

RunbookAI supports custom Handlebars templates stored in:

.runbookai/templates/

Use templates when your team wants a specific runbook, PR, postmortem, or changelog format.

Storage and outputs

Local session data.runbookai/ stores captured session records locally in the repository.
Generated docsdocs/runbooks/ contains generated Markdown artifacts.
RedactionBasic secret redaction is applied to captured diffs and outputs.
Git awarenessChanged files and Git diff context are captured to make handoffs more complete.
