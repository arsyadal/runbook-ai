# Runbook v0.1.0

First MVP release of Runbook — a local-first CLI that turns AI-assisted coding sessions into reusable engineering documentation.

## Highlights

- Record debugging/development sessions locally.
- Capture wrapped commands, exit codes, duration, stdout/stderr previews, and detected errors.
- Add manual notes for findings, decisions, risks, todos, failed attempts, and root causes.
- Capture Git snapshots and changed files.
- Generate Markdown runbooks, changelog drafts, postmortems, and GitHub PR descriptions.
- Export session data as JSON or Markdown.
- Search previous sessions.
- Use MCP server support so AI agents can query saved sessions.
- Use shell integration for Bash, Zsh, and PowerShell.
- Run `runbook doctor` to diagnose local setup issues.
- Generate AI-assisted summaries with OpenAI, Gemini, or local Ollama.

## Install

Download the binary for your platform from the release assets.

Or install from this repository:

```bash
cargo install --git https://github.com/arsyadal/runbook-ai
```

Or install from a local checkout:

```bash
cargo install --path .
```

## Quickstart

```bash
runbook init
runbook start "Fix login 401 error"
runbook exec "npm test"
runbook note --type root-cause "JWT secret was missing in test environment."
runbook exec "npm run build"
runbook stop
runbook generate all
```

Generated docs are written to:

```txt
docs/runbooks/
```

Session data is stored locally in:

```txt
.runbook/
```

## Shell integration

```bash
source <(runbook shell-hook zsh)
source <(runbook shell-hook bash)
```

```powershell
runbook shell-hook powershell | Invoke-Expression
```

## AI provider configuration

- `OPENAI_API_KEY` → OpenAI, default model `gpt-4o`
- `GEMINI_API_KEY` → Gemini, default model `gemini-1.5-pro`
- no API key → local Ollama, default model `llama3`

Override model:

```bash
RUNBOOK_MODEL=llama3.1 runbook generate runbook --ai
```

## Notes

Runbook is local-first by default. It does not require an API key for normal recording and documentation generation.
