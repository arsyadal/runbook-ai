Built for the messy middle of AI-assisted engineering.

The final diff does not explain the investigation. Runbook preserves the trail that normally disappears into chat history and terminal scrollback.

01

Agent handoff

When a model hits a context limit, generate a Next-Agent Brief so the next AI tool can continue without a long re-explanation.

02

Incident debugging

Capture commands, errors, failed attempts, root cause, and verification steps while fixing production issues.

03

PR descriptions

Turn the session into a reviewer-friendly summary of what changed, why it changed, how it was tested, and what risks remain.

Record the work. Generate the memory.

Runbook sits alongside your existing tools. Keep using your terminal and AI agent; Runbook turns the session into durable engineering artifacts.

$ runbook start "Fix login 401"
$ runbook exec "npm test"
✕ Auth test failed: missing JWT secret
$ runbook note --type root-cause "test env missing JWT_SECRET"
$ runbook exec "npm run build"
✓ build passed
$ runbook generate all

Without Runbook, context is fragile.

With Runbook, the repo becomes the source of truth for what happened during an AI-assisted coding session.

Quickstart

Start preserving your next debugging session.

Install from source, start a session, run commands, add notes, and generate reusable documentation.

cargo install --git https://github.com/arsyadal/runbook-ai
runbook init
runbook start "Fix payment 500"
runbook exec "npm test"
runbook note --type root-cause "JWT secret missing"
runbook generate all

Do not let your AI session disappear when the chat ends.

Capture the fix, preserve the reasoning, and continue anywhere with durable engineering memory.
